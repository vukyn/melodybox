melodybox — Offline MP3 Player

ESP32-classic desk MP3 player. No WiFi. 5 buttons drive playback of tracks on the YX6300 module's SD card; an SSD1306 OLED shows the now-playing screen. Feature-doc standard: BOM → wiring → pin map → data flow → sim.

1 · Usecase

Sits on the desk, fully offline. Press PLAY/PAUSE to toggle, NEXT/PREV to change track (wraps at TRACK_COUNT), VOL+/VOL- to set volume 0–30. The OLED shows melodybox + current Track N + play/pause state + volume. Boots paused — the user presses play to start.

3 · Wiring diagram

3V3 → OLED VCC VIN (~5V) → YX6300 VCC GND → GND (common) GPIO21 → OLED SDA GPIO22 → OLED SCL GPIO16 (RX) ← module TX GPIO17 (TX) → module RX GPIO32/33/25/26/27 → button → GND

Power split: YX6300 on VIN (5V), OLED on 3V3. Powering the YX6300 from 3V3 under-powers its SD card and DAC → phantom SD insert/remove events (the flip-flop bug). Keep the OLED on 3V3 so its I2C SDA/SCL stay at the ESP32's 3.3V logic level.

GPIO21/22 = ESP32-classic HW-default I2C. UART2 (Serial2) is remapped to GPIO16/17 — note the crossover: ESP32 RX ← module TX, ESP32 TX → module RX. All 5 button pins are pull-up capable, non-strapping, non-input-only. OLED I2C addr 0x3C. Change any pin in include/config.h. START_VOLUME 18, TRACK_COUNT 20.

One loop() tick (~40ms, ~25fps): checkButtons() → mutate state + fire the matching YX5300 command → marquee += 1 → render() redraws the OLED. Every action keeps local state and the module in sync (e.g. nextTrack() = index wrap + playIndex() + playing=true).
